# Commands Reference

Complete guide to all Role Reactor Bot commands with examples and usage tips.

## Role Message Commands

### `/setup-roles`

Create a new role selection message with emoji reactions.

**Parameters:**
- `title` (required) - Title for the role message (max 256 characters)
- `description` (required) - Description text (max 2000 characters)
- `roles` (required) - Role-emoji pairs in format: `emoji:role` (max 4000 characters)
- `color` (optional) - Embed color as hex code or predefined choice

**Examples:**

<Tabs items={['Basic', 'With Categories', 'Advanced']}>
  <Tab value="Basic">
    ```
    /setup-roles 
    title:"Server Roles" 
    description:"Choose your interests!"
    roles:"🎮:Gaming,📚:Books,🎵:Music"
    ```
  </Tab>
  
  <Tab value="With Categories">
    ```
    /setup-roles 
    title:"Community Roles"
    description:"Pick your roles by category"
    roles:"#Interests
    🎮:Gaming
    📚:Reading
    #Notifications
    📢:News
    🎉:Events"
    ```
  </Tab>
  
  <Tab value="Advanced">
    ```
    /setup-roles 
    title:"🌟 Server Roles" 
    description:"**Welcome!** React below to get roles.\n\n*Multiple selections allowed.*"
    roles:"#Gaming 🎮
    🖥️:PC Gaming
    🎮:Console Gaming
    #Creative 🎨
    🎨:Digital Art
    🎵:Music"
    color:"#0099ff"
    ```
  </Tab>
</Tabs>

### `/update-roles`

Update an existing role message.

**Parameters:**
- `message_id` (required) - ID of the message to update
- `title` (optional) - New title for the message
- `description` (optional) - New description
- `roles` (optional) - New role configuration
- `color` (optional) - New embed color

**Example:**
```
/update-roles 
message_id:123456789012345678
title:"Updated Server Roles"
roles:"🎮:Gaming,📚:Books,🎵:Music,🎨:Art"
```

### `/delete-roles`

Delete a role message and remove all associated reactions.

**Parameters:**
- `message_id` (required) - ID of the message to delete

**Example:**
```
/delete-roles message_id:123456789012345678
```

### `/list-roles`

List all role-reaction messages in the current server.

**Example:**
```
/list-roles
```

## Temporary Role Commands

### `/assign-temp-role`

Assign temporary roles to users that automatically expire.

**Parameters:**
- `users` (required) - Comma-separated user IDs or mentions
- `role` (required) - The role to assign temporarily
- `duration` (required) - How long the role lasts (e.g., 1h, 2d, 1w, 30m)
- `reason` (optional) - Reason for assignment

**Examples:**
```
/assign-temp-role 
users:@User1,@User2 
role:@VIP 
duration:7d 
reason:"Event winners"

/assign-temp-role 
users:123456789012345678 
role:@Muted 
duration:30m 
reason:"Timeout for spam"
```

### `/remove-temp-role`

Remove a temporary role before it expires.

**Parameters:**
- `user` (required) - User to remove the role from
- `role` (required) - The temporary role to remove
- `reason` (optional) - Reason for removal

**Example:**
```
/remove-temp-role 
user:@Username 
role:@VIP 
reason:"Early removal requested"
```

### `/list-temp-roles`

List all active temporary roles in the server.

**Parameters:**
- `user` (optional) - Show temporary roles for specific user only

**Examples:**
```
/list-temp-roles
/list-temp-roles user:@Username
```

## Role Format Guide

### Basic Role Format

Simple comma-separated list:
```
🎮:Gaming,📚:Books,🎵:Music,🎨:Art
```

### Category Format

Use `#Category Name` for headers:
```
#Interests
🎮:Gaming
📚:Books
#Notifications  
📢:News
🎉:Events
```

### Advanced Multi-Category

Multiple categories with organized structure:
```
#Gaming 🎮
🖥️:PC Gaming
🎮:Console Gaming
📱:Mobile Gaming
#Creative Arts 🎨
🎨:Digital Art
📸:Photography
🎵:Music Production
#Notifications 📢
📢:Important News
🎉:Fun Events
📰:Weekly Updates
```

## Duration Format

For temporary roles, use these duration formats:

    - `45m` - 45 minutes

  <Tab value="Hours">

    - `12h` - 12 hours

## Command Tips

<Callout type="info">
**Pro Tips for Better Role Messages:**

- Use clear, descriptive role names without special characters
- Keep role lists manageable (5-12 roles per message works best)
- Use categories to organize related roles together
- Test your role messages after creating them
- Use `/list-roles` to get message IDs for updates
</Callout>

## Error Handling

**Common Issues:**
- **"Role not found"** - Check role name spelling exactly
- **"Missing permissions"** - Ensure you have Manage Roles permission
- **"Cannot assign role"** - Bot's role must be above target roles
- **"Invalid duration"** - Use proper format like "1h", "30m", "7d"

**Getting Message IDs:**
1. Enable Developer Mode in Discord settings
2. Right-click on the role message
3. Select "Copy Message ID"
4. Use this ID in update/delete commands
